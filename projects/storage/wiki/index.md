# Knowledge Wiki

## Concepts

- **[[annualized-failure-rate|Annualized Failure Rate]]** — AFR measures annual failure probability of storage devices; VIHR SSDs achieve 0.09% vs standard 0.35%
- **[[failure-prediction|Failure Prediction]]** — Predictive capability using telemetry data including IO timeout, latency spikes, and data miscompare to anticipate SSD failures
- **[[fault-system-resiliency|Fault System Resiliency]]** — Storage device capability that detects and recovers from firmware panics, crashes, or hard hangs through non-invasive corrective actions
- **[[mtbf|MTBF]]** — Mean Time Between Failures - industry standard metric; VIHR achieves 10M hours vs standard 2.5M hours
- **[[nvme-telemetry-log-page|NVMe Telemetry Log Page]]** — NVMe standardized telemetry mechanism including Data Areas 1, 2, 3 for device health monitoring, failure prediction, and root cause analysis
- **[[panic-recovery-mechanism|Panic Recovery Mechanism]]** — Storage device recovery process using 0xC1 Error Recovery Log Page, AER, and CSTS.CFS bit to gracefully handle fatal failures
- **[[vihr-ssd|VIHR SSD]]** — Vertically Integrated High Resilience SSD - Microsoft initiative enhancing cloud storage reliability through standardized OCP v2.5+ specifications
