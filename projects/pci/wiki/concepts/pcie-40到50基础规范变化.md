# PCIe 4.0 到 5.0 基础规范变化

PCI Express 5.0（PCIe 5.0）是 PCI Express 规范的下一代版本，相比 PCIe 4.0 在数据传输速率、物理层信号完整性和测试要求等方面均进行了显著升级。本页面汇总 PCIe 4.0 与 PCIe 5.0 基础规范之间的关键差异。 ^[presentation-title-goes-here.md]

## 核心参数对比

下表列出了 PCIe 4.0 与 PCIe 5.0 基础规范的核心参数变化： ^[presentation-title-goes-here.md:4-4]

| 参数 | PCIe 4.0 | PCIe 5.0 |
|------|-----------|----------|
| **数据速率** | 16 GT/s | 32 GT/s |
| **信道损耗（Rx 测试范围）** | - (27 至 30) dB @ 8GHz | - (34 至 37) dB @ 16GHz |
| **附加卡损耗** | 8dB @ 8GHz | 9.5dB @ 16GHz |
| **参考 CTLE** | 2 极点；1 零点；直流增益范围 (-6 至 -12) dB | 4 极点；2 零点；直流增益范围 (-5 至 -12) dB |
| **参考 DFE** | 2 抽头 | 3 抽头 |
| **眼宽（Rx 测试）** | 18.75 ps | 9.375 ps |
| **眼高（Rx 测试）** | 15 mV | 15 mV |
| **Rx 抖动容限** | CC 与 SRIS 在 16GT/s 下使用独立曲线 | CC 与 SRIS 在 16GT/s 和 32GT/s 下使用统一曲线 |
| ** lane 边际检测** | 仅用于时序 | 用于时序和电压 |
| **参考时钟高频抖动限值** | ≤ 500 fs | ≤ 150 fs |

## 关键变化详解

### 数据速率翻倍

PCIe 5.0 将单通道数据速率从 16 GT/s 提升至 32 GT/s，实现了相比 PCIe 4.0 的翻倍提升。这一提升直接增加了对信道带宽和信号完整性的要求，体现在更高的信道损耗测试范围（从 8GHz 提升至 16GHz）上。 ^[presentation-title-goes-here.md:4-4]

### 接收端均衡增强

为应对更高的信道损耗和更快的数据速率，PCIe 5.0 显著增强了接收端均衡能力：

- **CTLE（连续时间线性均衡器）**：从 2 极点/1 零点配置升级为 4 极点/2 零点配置，直流增益范围从 (-6 至 -12) dB 调整为 (-5 至 -12) dB
- **DFE（判决反馈均衡器）**：抽头数从 2 增加到 3，以更好地补偿码间干扰 ^[presentation-title-goes-here.md:4-4]

### 眼图参数调整

由于数据速率翻倍，PCIe 5.0 的接收端眼图参数进行了相应调整：

- **眼宽**：从 18.75 ps 降至 9.375 ps（缩短一半）
- **眼高**：保持 15 mV 不变 ^[presentation-title-goes-here.md:4-4]

### 抖动容限统一

PCIe 5.0 在 Rx 抖动容限方面进行了简化，将原本针对 CC（公共时钟）和 SRIS（分离参考时钟独立展频）分别定义的独立曲线合并为统一曲线，适用于 16GT/s 和 32GT/s 两个速率。 ^[presentation-title-goes-here.md:4-4]

### 参考时钟要求更严格

PCIe 5.0 对参考时钟的高频抖动限值提出了更严格的要求，从 PCIe 4.0 的 ≤ 500 fs 降低至 ≤ 150 fs，降幅达 70%。 ^[presentation-title-goes-here.md:4-4]

## 链路均衡旁路

PCIe 5.0 规范引入了**链路均衡旁路**（Link Equalization Bypass）功能，作为可选特性： ^[presentation-title-goes-here.md:7-7]

- **直接训练模式**：支持从 2.5 GT/s 直接训练至 32 GT/s，跳过中间的 8GT/s 和 16GT/s 训练步骤
- **省时优势**：相比传统逐级训练方式，约可节省 200ms 的链路建立时间
- **全速跳过选项**：如果链路上的所有组件均支持，训练过程可完全跳过低速率步骤

## 突发错误预防

在 32 GT/s 速率下，突发错误（burst error）发生的概率增加，这与判决反馈均衡器的抽头比率有关。1010... 交替模式特别容易引发错误传播。PCIe 5.0 引入了以下防护机制： ^[presentation-title-goes-here.md:8-8]

- **预编码（Precoding）**：通过限制交替模式的出现次数来抑制突发错误
- **启用机制**：接收端可在 ≥ 32 GT/s 速率下通过训练序列（Training Sets）向发送端请求启用预编码

## 连接器和金手指优化

PCIe 5.0 的卡级规范（CEM）对连接器和金手指布局进行了优化： ^[presentation-title-goes-here.md:9-9]

- **仅允许 SMT（表面贴装技术）CEM 连接器**
- **优化接地过孔位置**：在焊盘两侧增加接地过孔
- **缩小焊盘尺寸**
- **减小金手指尺寸**
- **金手指下方增加接地平面**

金手指间距保持为 1.27mm（0.05 英寸）。 ^[presentation-title-goes-here.md:9-9]

## Lane 边际检测扩展

PCIe 5.0 将 lane 边际检测（Lane Margining）的应用范围从仅时序扩展至时序和电压两个维度，为系统提供了更全面的信号质量监控手段。 ^[presentation-title-goes-here.md:4-4]

## 规范发布状态

截至 2020 年 4 月，PCIe 5.0 相关规范的状态如下： ^[presentation-title-goes-here.md:3-3]

- **PCIe 5.0 基础规范**：1.0 版本已于 2019 年第二季度发布，定义了所有层级的芯片级行为
- **PCIe 5.0 CEM 规范**：0.7 版本正在开发中，定义了系统和附加卡层级的机电规范
- **PCIe 5.0 PHY 测试规范**：0.3 版本已获工作组批准，定义了发送端、Rx LEQ 及 PLL 带宽的电气合规测试
- **PCIe 4.0 所有规范**：1.0 版本已就绪，2020 年 8 月已启动集成商列表测试
